## readonly

```tsx
// Array
const fruits: string[] = ['🍅', '🍌'];
const scroes: Array<number> = [1, 3, 4];
function printArray(fruits: readonly string[]) {
  // string.push()  x
}
```

fruit배열안에는 토마토, 바나나가 들어있습니다.
이렇게 들어있는 배열에 타입을 정의는 2가지방법이있다.

string[] / number[]Array< number > 이런식으로 정의할수있다.
전달된 배열을 출력하는 함수가 있다면 이 함수는 주어진 데이터를 이용해서 무언가 출력할 수 있지만, 주어진 데이터를 변경하거나 업데이트할 수 없을 수있다. readonly를 사용하면 fruits는 데이터를 변경할 수 없고 오로지 읽기만 가능하다. 오브젝트의 불변성을 지켜준다.

#### Tuple

```tsx
// Tuple -> interface, type alias, class
let student: [string, number];
student = ['name', 123];
student[0]; // name
student[1]; // 123
const [name, age] = student;
```

튜플은 서로다른 타입의 배열을 가질 수 있게해준다.
student는 문자와 숫자타입을 정의한다.
0번째 인덱스는 문자
1번째 인덱스는 숫자를 읽어올 수 있다.
튜플을 사용하는 것을 권장하지않는다.
그 이유는 데이터에 접근할때, 인덱스처럼 접근하는 것은 가독성이 현저히 떨어진다. 배열안에 데이터를 직접확인해야 하기때문에 불편하다.
튜플대신에 오브젝트를 사용한다.
튜플을 사용하는 곳은 interface, type alias, class로 대체해서 사용한다.
